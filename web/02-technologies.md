# Список технологий с предназначением

## CI/CD
- автоматизация развертывания приложения, с тестированием и прочими этапами (любыми)
- "непрерывная интеграция и непрерывная доставка"

Классические сервисы: *GitLab*, *Jenkins*, *GitHub Actions*, ...

Пишется на `yaml` ну и прочее.

## Секреты
- понимается пароли от базы данных, другие пароли
- обычно переменные окружения

Сервисы: *Vault*, *GitLab*, ...

## Хранение Docker образов
- при CI собирается докер контейнер, его нужно хранить где-то
- нужно например чтобы откатываться по версиям (в Dockerfile в частности можно указать версию), можно храннить несколько образов: продакшн, разработка

Сервисы: (Container Registry) *Docker HUB*, *GitLab*, *Harbor*, *GitHub*

## Kubernetes
- например, если нужно быстро масштабироваться

Сервисы: *LENS*, *Dashboard Kubernetes*, *Argo CD* (argo это хорошо)

## Монитоирнг и логирования
- метрики приложения (CPU, mem, ...)

Сервисы: *Graphana*, *Graphana Loki* (можно настроить чтобы в ТГ приходили оповещения о проблемах), ELK (ёлка - *elasticsearch*, *logstash*, *kibana*)

## Веб сервера
- избежание конфигурирования nginx, haproxy.
- берут на себя часть ответственности за запросы 

Сервисы: *nginx*, *haproxy*


Цикл производства веб приложения
1) Исследование и анализ
    * анализ рынка, UX-исследования...
2) Проектирование:
    * выбор архитектуры: монолитная либо микросервисная
    * **SPA** (single-page application)
    * **MPA**  (многостраничные - сложнее, и у фронта, и у бэка должны быть фреймворки)
    * **PWA** (progressive web applications) - можно ставить на главный экран
