# Разное про WEB

## Model-View-Controller

Схема разделения логики приложения на 3 инкапсулированных друг от друга компонента, связанные друг с другом интерфейсом.

* *Model* - внутренний интерфейс приложения. То есть вся внутренняя логика, например работа с БД.
* *View* (представление) - получает данные от модели и отображает их пользователю.
* *Controller* (контроллер) - реагирует на действия пользователя и оповещает об этом модель.

## Термины
* **DTO** - Data Transfer Object
* **JWT** - JSON Web Token
* **CRUD** - Create, Read, Update, Delete
* **gRPC** (акроним gRPC Remote Procedure Calls) - фреймворк для удаленного вызова процедур (RPC) - процедуры вызываемые в другом адресном пространстве (например, на удаленных узлах)
* **Web API** - архитектура обмена данными посредством HTTP/HTTPS. Обычно подразумевает RESTful API, RPC, взаимодесствие с помощью JSON.
* **REST**:
  * GET - получить
  * POST - отправить/добавить
  * PUT - изменить
  * DELETE - удалить
* Масштабирование - действия для увеличения пропускной способности и мощности приложения
  * Вертикальное (Scale Up/Vertical) - увеличение мощностей текущего железа (память, количество процессоров и пр.)
  * Горизонтальное (Scalr Out/Horisontally) - увеличение количества железа, увеличение количества серверов.
